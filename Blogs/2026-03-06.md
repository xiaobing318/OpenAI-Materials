# 2026-03-06

## 1. 概述

过去 24 小时内，OpenAI 对开发者最值得关注的更新主要集中在三件事：第一，官方发布了 **GPT-5.4**，并明确已在 ChatGPT（以 GPT-5.4 Thinking 形式）、API 与 Codex 同步可用，且强调了 computer-use、工具检索与长上下文能力；第二，官方 SDK **openai-python v2.25.0** 与 **openai-node v6.26.0** 在同一天发版，变更点直接出现 `gpt-5.4`、`tool search tool` 与 `computer tool`，对 API 集成方是立即可感知的接口层信号；第三，**openai/codex v0.111.0** 发布，包含默认 Fast mode、插件发现与 app-server v2 MCP 交互改进，对 Codex App/CLI 与自动化工作流有直接影响。此外，状态页在窗口内出现多起已恢复事件，提示生产环境仍需保留重试、降级与观测策略。

## 2. 官方发布与公告

- **Introducing GPT-5.4**
  - 发布时间：2026-03-05（来源页仅给出日期，来源时区未明确）
  - 来源：https://openai.com/index/introducing-gpt-5-4/
  - 要点：
    - 官方确认 GPT-5.4 已在 ChatGPT、API、Codex 上线，并同时提供 GPT-5.4 Pro。
    - 强调这是首个具备原生高水平 computer-use 能力的通用模型，面向代理化专业工作流。
    - 提到支持更大上下文（文中写到最高 1M tokens）与更高 token 效率，意味着复杂任务成本/延迟有望下降。

- **Introducing ChatGPT for Excel and new financial data integrations**
  - 发布时间：2026-03-05（来源页仅给出日期，来源时区未明确）

  - 要点：
    - 发布 ChatGPT for Excel（Beta），可在表格内直接生成、更新与分析模型。
    - 文中明确 GPT-5.4（GPT-5.4 Thinking）已用于该场景，并给出内部金融任务表现提升数据。
    - 对企业管理员给出默认关闭与按角色开启机制，提示组织治理和权限配置是落地前置步骤。

- **OpenAI Status：3 月 5 日多起事件均已恢复**
  - 发布时间：2026-03-05（状态页事件时间；页面未明确显示时区名称）
  - 来源：https://status.openai.com/history
  - 要点：
    - 当日记录了 ChatGPT 发送消息异常、Realtime API（EU）错误率升高、API 错误率等事件。
    - 当前状态均为 “All impacted services have now fully recovered”。
    - 对线上业务意味着：仍应保留幂等重试、超时与回退模型策略，并持续订阅状态页。

## 4. Codex 产品与工程效率动态

- **openai/codex v0.111.0 发布**
  - 发布时间：2026-03-05 19:12（GitHub 页面展示时间，时区未明确）
  - 来源：https://github.com/openai/codex/releases
  - 要点：
    - Fast mode 默认开启，且 TUI 头部可见 Fast/Standard 状态，直接影响交互速度与使用习惯。
    - 提升插件与技能发现（会在会话启动时告知已启用插件），有利于多工具自动化任务命中率。
    - app-server v2 将 MCP elicitation 变成结构化请求/响应，降低客户端集成复杂度。
    - `codex resume` 相关状态保留问题被修复，减少长任务续跑时的上下文损坏风险。

- **GPT-5.4 在 Codex 可用（官方声明）**
  - 发布时间：2026-03-05（来源页仅给出日期，来源时区未明确）
  - 来源：https://openai.com/index/introducing-gpt-5-4/
  - 要点：
    - 官方明确 GPT-5.4 覆盖 Codex 场景，并强调编码与工具协同能力。
    - 对 Codex 用户而言，意味着“模型能力升级 + CLI/App 版本升级”需要配套评估。
    - 建议把现有自动化任务做一轮基准测试（成功率、耗时、token 成本）。
